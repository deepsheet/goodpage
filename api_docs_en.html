🚀 GoodPage API Documentation

Transform plain text into beautiful HTML web pages with one click

📖 Overview

GoodPage provides powerful text-to-HTML API services that can automatically convert plain text into beautiful web pages with rich visual effects. Supports multiple style templates, intelligent typesetting, and custom requirements.

✨ Smart Beautification

AI automatically analyzes content and intelligently matches the best layout style and visual elements

🎨 Diverse Templates

17+ professional design templates to meet display needs for different scenarios

⚡ Fast Response

Usually completes conversion within 3-10 seconds, supports high concurrent calls

🔗 Flexible Output

Can return HTML code or directly generate accessible web page links

🌐 API Endpoint

POST https://p.isheet.net/api/text-to-html

📝 Request Parameters

Request Headers

Content-Type: application/json

Request Body (JSON Format)

Parameter Type Required Default Description
content string Yes - Text content to convert
title string No AI Generated Article title (if not provided, AI will auto-generate)
content_strategy string No strict Expansion mode:
strict - Strictly follow original
interpret - Interpret & optimize
expand - Expand & enrich
extra_requirements string No Empty Additional requirements, describe formatting needs in natural language, e.g., "Use blue theme", "Add tables to display data"
response_type string No html Response type:
html - Return HTML code
url - Return webpage URL
style string No AI Selected Specify style template code, e.g., minimalist, premium_bw, tech_blue, etc.

🔧 Usage Examples

Python Example

import requests

# API 端点
url = "https://p.isheet.net/api/text-to-html"

# 请求数据
payload = {
    "content": """Artificial intelligence is transforming the world
    
With technological advancements, artificial intelligence has penetrated into every aspect of our lives. From smartphones to autonomous vehicles, AI technology is reshaping our way of life.

Future outlook:
1. A smarter personal assistant
2. Innovation in medical diagnosis
3. Personalized education
4. "Industrial automation",
    "title": "Artificial Intelligence Changes Life",
    "response_type": "url"
}

# Send Request
response = requests.post(url, json=payload)

# Check response
if response.status_code == 200:
    result = response.json()
    if result["status"] == "success":
        article_url = result["full_url"]
        print(f"✅ Web page generated successfully!")
        print(f"🔗 Access link: {article_url}")
    else:
        print(f"❌ Generation failed: {result['message']}")
else:
    print(f"❌ Request failed, status code: {response.status_code}")

JavaScript Example

async function textToHtml() {
    const url = 'https://p.isheet.net/api/text-to-html';
    
    const payload = {
        content: `Artificial intelligence is changing the world

With the advancement of technology, artificial intelligence has penetrated into every aspect of our lives. `,
        title: 'Artificial Intelligence Changes Life',
        response_type: 'url'
    };
    
    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(payload)
        });
        
        const result = await response.json();
        
        if (result.status === 'success') {
            console.log('✅ Web page generated successfully!');
            console.log('🔗 Access link:', result.full_url);
        } else {
            console.error('❌ Generation failed:', result.message);
        }
    } catch (error) {
        console.error('❌ Request exception:', error.message);
    }
}

textToHtml();

❓ Frequently Asked Questions

Q: Why am I getting a 403 error?

A: Your IP address quota has been exhausted. Each IP has 50 free call quota. After exceeding, please contact customer service to upgrade.

Q: How to get permanent links for articles?

A: Use the response_type: "url" parameter, the API will return the complete webpage access link.

Q: Can the generated HTML be used offline?

A: Yes, all styles are inline, no external resources needed, can be fully used offline.

Q: What languages are supported?

A: Supports Chinese, English and other languages. The API will automatically detect the text language and apply corresponding typesetting rules.

Q: How to handle long text?

A: It is recommended not to exceed 10,000 characters per request. For ultra-long text, you can process it in segments or contact customer service for optimization solutions.
